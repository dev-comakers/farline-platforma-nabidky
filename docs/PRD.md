## 1. Контекст и цель

Farline Living поставляет премиум-сантехнику архитекторам (Lefroy & Brooks, CoalBrook, Radomonte, Victoria + Albert, Cielo, Viega и др.). Сейчас Филип вручную собирает коммерческие предложения (КП) в Google Sheets и шлёт статичные PDF.

**Цель:** веб-платформа «каталог → конструктор КП → публичная ссылка → комментарии архитектора → экспорт PDF/Excel», которая сокращает подготовку КП с 60+ минут до ~5 и делает коммуникацию интерактивной.

---

## 2. Пользователи и роли

Платформа **многопользовательская** (до ~5 внутренних пользователей Farline). Авторизация обязательна для всей внутренней зоны. Публичная страница КП — без авторизации.

| Роль | Может | Не может |
|---|---|---|
| **admin** | всё; **создавать/редактировать/удалять пользователей**; задавать пароли; редактировать определения полей категорий | — |
| **manager** | создавать/редактировать КП и продукты, шарить, экспортировать, читать комментарии | **удалять КП**; управлять пользователями |
| **architect (внешний, без аккаунта)** | открыть публичную ссылку, смотреть КП read-only, скачать PDF/Excel, оставить комментарий, открыть карточку продукта и скачать техлист | всё остальное |

> Разница admin/manager минимальна: **admin управляет пользователями и может удалять КП; manager — нет.** Детали прав — в `TECH-SPEC.md`.

## 4. Функциональные требования (acceptance-критерии)

> Формат: каждая фича — чек-лист. Готово = все пункты ✔. Czech-строки UI сохраняем как есть.

### FR-1. Авторизация и пользователи
- [ ] Логин-страница (email + пароль), JWT в httpOnly cookie.
- [ ] Middleware защищает всё, кроме `/nabidka/[id]`, логина и статики.
- [ ] Сессия ~7 дней; logout чистит cookie.
- [ ] Экран «Uživatelé» доступен **только admin**: список, создать пользователя (email, имя, роль, пароль), сбросить пароль, удалить.
- [ ] Manager не видит экран пользователей и не может удалять КП (кнопка скрыта + проверка на API).
- [ ] Первый admin создаётся seed-скриптом из ENV.
- [ ] Самостоятельной регистрации нет. Email-сброса пароля нет (пароль задаёт admin).

### FR-2. Дашборд (`/`)
- [ ] 4 метрики: Celkem nabídek, Rozpracované, Odeslané, Nové komentáře (Geist Mono).
- [ ] «Poslední nabídky» — таблица последних КП с переходом в детали.
- [ ] «Aktivita» — таймлайн (новый комментарий / КП отправлена / создана).
- [ ] Быстрые действия: «Nová nabídka» **и «Otevřít katalog»**.
- [ ] Все данные из БД, не из памяти.

### FR-3. Каталог (`/katalog`)
- [ ] Grid/list, поиск (код, название, бренд, декор), фильтры бренд/декор/тип.
- [ ] Карточка: фото (или заглушка-иконка по категории), код, название, бренд, декор, цена без/с НДС по настройке.
- [ ] Кнопка «Přidat produkt» → форма (см. FR-4).
- [ ] Кнопка «Importovat» → импорт (см. FR-6).
- [ ] Клик по продукту → детальная карточка (см. FR-5).

### FR-4. Создание/редактирование продукта
- [ ] Базовые поля: kód*, název*, značka, dekor, typ (категория), cena, měna (CZK/USD/EUR).
- [ ] **Параметры по категории**: при выборе категории форма подтягивает её поля (текст/число/select с опциями) из определений категории (admin-editable, см. FR-9).
- [ ] Загрузка фото: drag&drop, JPG/PNG/WebP, превью, замена/удаление; сохраняется на сервере.
- [ ] Загрузка техлиста: 1 PDF на продукт, замена/удаление.
- [ ] Валидация: kód и název обязательны.
- [ ] Редактирование любого поля; удаление с подтверждением.

### FR-5. Детальная карточка продукта (модалка)
- [ ] Открывается из каталога и из публичной страницы КП.

## 5. Бизнес-правила

### 5.1 Деньги и НДС
- Валюта задаётся **на уровне КП** (не позиции). Поддержка **CZK, USD, EUR**. Конвертации между валютами нет.
- Скидка — процент на позицию (0–100).
- **НДС: 21 %, фиксированная.** По умолчанию показывается; на уровне КП есть toggle «скрыть НДС».
- Вся денежная математика — только в `lib/calculations.ts` (subtotal → sleva → základ → DPH → s DPH). Рендер чисел — Geist Mono + `tabular-nums`.

### 5.2 Статусы КП
`rozpracovaná → odeslaná → okomentovaná → potvrzená` (цвета — `DESIGN-SYSTEM.md`).
- Шаринг → авто «odeslaná».
- Новый комментарий → авто «okomentovaná».
- «potvrzená» ставит пользователь вручную.

### 5.3 Публичная ссылка
- Непредсказуемый UUID-токен `shareId`, **отдельный от внутреннего id КП** (маршрут `/nabidka/[shareId]`), бессрочная. Это единственный контроль доступа к публичной странице.
- Возможность «закрыть/деактивировать» ссылку — **не сейчас**, заложить флаг `shareEnabled` на будущее (`TECH-SPEC.md`).

### 5.4 Статусы позиции (независимы от статуса КП)
Цепочка: `nepotvrzená → Potvrzeno → Objednáno → Získáno`.
- Отмечает Farline (admin/manager), не архитектор.
- Видны архитектору на публичной странице (read-only).

## 6. Нефункциональные требования
- LCP < 2 c; интеракции (пересчёт, добавление позиции) < 100 мс; PDF < 5 c; Excel < 3 c.
- Десктоп (1024+) и планшет (768+) полностью; публичная страница КП — адаптив до 320px.
- Браузеры: Chrome/Safari/Firefox/Edge — последние версии.
- Доступность: keyboard-навигация, ARIA на формах, контраст ≥ 4.5:1.
- Безопасность: непредсказуемые UUID, без чувствительных данных в URL, санитизация вводимого (комментарии, имена), пароли — хэш (bcrypt/argon2), JWT в httpOnly cookie. Подробнее — `TECH-SPEC.md`.
