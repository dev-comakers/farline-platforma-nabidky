# IMPLEMENTATION-PLAN — Farline Nabídky

**Версия:** 1.0 · **Дата:** 2026-05-21
План разбит по **блокам/категориям** (не по дням). Каждый блок = один или несколько PR. Идём сверху вниз: блоки 1–3 — фундамент, дальше функциональность опирается на него.

**Правила работы (см. `TECH-SPEC.md` §10):**
- Один блок → отдельная ветка (`feat/...`) → PR с описанием → CI зелёный → ревью Artem → мёрж.
- Перед PR: `lint`, `typecheck`, `build`, `test` локально.
- Дизайн заморожен (`DESIGN-SYSTEM.md`). Стек зафиксирован (`TECH-SPEC.md`). Скоуп — `PRD.md` (раздел «Вне скоупа» = запрет).
- Если требование неясно или дрейфует во «Вне скоупа» — остановись и спроси Artem.

Легенда зависимостей: ➡️ «зависит от».

---

## ВАЖНО: исходный код демо включён в архив

Это **не greenfield**. В архиве лежит рабочий клиентский прототип (Next.js 16 + React 19 + Tailwind v4), UI которого **согласован клиентом и заморожен** (`DESIGN-SYSTEM.md`). Твоя задача в большинстве блоков — **не строить с нуля, а подключить готовый UI к БД и добавить недостающее**. Переписывать существующие экраны/компоненты ради «по-своему» — нельзя.

## Поправки по итогам ревью (применять в блоках)

Детали и обоснование — `TECH-SPEC.md §12`. Кратко по блокам:

- **Блок 0:** создать `.github/workflows/ci.yml` (`npm ci → lint → typecheck → build → test`) и скрипты в `package.json` (`typecheck`, `test`); добавить `output: "standalone"` в `next.config.ts`; перенести Satoshi на `next/font/local`. Тесты unit-only.
- **Блок 1:** деньги — **`Decimal`**, не `Float`; `Offer.shareId` отдельно от `id`; `Product.code @unique`; `@@index` на FK и hot-колонки; `OfferItem.unitPriceSnapshot`; `Offer.vatRate`; **`EUR` в enum/типы с первого дня**; seed идемпотентный (`upsert` админа).
- **Блок 2:** файл — **`proxy.ts`**, не `middleware.ts` (Next 16); авторизация ролей — в каждом хендлере (`requireAdmin`), не только в proxy; throttle логина; JWT-secret ≥ 32 байта, пин алгоритма; httpOnly+Secure+SameSite=Lax.
- **Блок 3:** **удалить** `app/share/[id]` (включая `layout.tsx` со `StoreProvider`); починить бейдж в `Sidebar.tsx` (он импортит `useStore` — заменить на серверные данные, иначе сборка падёт); контракт автосейва (debounce 1000 мс, индикатор, retry); комментарии в детали КП мигрировать на чтение из БД (без отправки — это Блок 8).
- **Блок 4:** отдача файлов — решение из `§12.5` (Caddy для фото, `[auth]`-роут для техлиста); валидация по magic bytes; защита от path traversal; `images: { unoptimized: true }` + `<img>`.
- **Блок 6:** добавить расчёт НДС в `lib/calculations.ts` (`VAT_RATE`, `vatAmount`, `totalWithVat` в `OfferSummary` — сейчас их нет); **EUR** в `lib/types.ts` и `formatCurrency` + в `CurrencyToggle`.
- **Блок 7:** `share` идемпотентен; публичный ответ — **белый список полей** (`offerPublicSelect`, без `internalNote`/`authorEmail`); поиск по `shareId`.
- **Блок 8:** конкретный механизм rate-limit; HTML-экранирование полей в письме Resend; фиксированные `from/to`; эндпоинт `…/comments/read`.
- **Блок 10:** импорт — **два шага** (`/preview` и `/commit`); Zod-валидация.
- **Блок 12:** создать `Dockerfile` + `docker-compose.yml` (restart/healthcheck/depends_on healthy/только Caddy наружу) + `Caddyfile` (+security-заголовки) + `entrypoint.sh` (`migrate deploy` → `node server.js`); build-time vs runtime `DATABASE_URL`; конкретные бэкап+restore команды; предупреждение про `down -v`; шпаргалка операций.
- **Сквозное (§5/§12.3):** единый формат ошибки + коды статусов; пагинация списков; аннотации доступа `[public]/[auth]/[admin]`; полный CRUD путей для полей категорий.
