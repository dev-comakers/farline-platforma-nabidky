# Onboarding — Farline Nabídky

Привет! Ты подключаешься к разработке **Farline Nabídky** — веб-платформы для премиум-поставщика сантехники Farline Living. Этот файл — точка входа. Прочитай его целиком, потом документы из `docs/` по порядку.

---

## 1. Что это за продукт

Платформа заменяет ручную сборку коммерческих предложений (КП) для архитекторов в Google Sheets. Поток:
**каталог продуктов → конструктор КП → публичная ссылка для архитектора → комментарии → экспорт PDF/Excel.**

Внутренние пользователи (Farline, роли admin/manager) собирают КП. Архитектор открывает публичную ссылку без логина, смотрит read-only КП, скачивает PDF/Excel и оставляет комментарий → Farline получает письмо.

---

## 2. Текущее состояние кода ⚠️

В архиве — **рабочий клиентский прототип**: Next.js 16 (App Router) + React 19 + Tailwind v4. UI **согласован клиентом и заморожен**.

**НО:** данные сейчас живут в памяти (`lib/store.tsx`) и сбрасываются при перезагрузке. Нет БД, авторизации, реальной почты и импорта. Твоя работа — превратить прототип в продакшен по `docs/`, **сохранив визуал** и **переиспользуя существующий UI/логику**, а не переписывая их.

## 4. Правила (не обсуждаются)

- **Дизайн заморожен.** Не менять цвета/шрифты/компоненты/анимации. Только подключать данные. Чек-лист — `DESIGN-SYSTEM.md §9`.
- **Стек зафиксирован.** Postgres+Prisma, JWT(httpOnly)+middleware, Docker Compose+Caddy на Hetzner, локальный том для файлов, Resend. Не подменять компоненты стека.
- **Скоуп.** Раздел «Вне скоупа» в `PRD.md §9` — это запрет. Дрейфует туда — остановись и спроси.
- **Процесс.** Один блок плана = одна ветка `feat/...` = один PR. Перед PR: `lint + typecheck + build + test`. CI должен быть зелёным. Описание PR обязательно (что сделано, какие критерии закрыты, как проверить).
- **Язык.** UI-строки — чешский (сохранять существующие). Код/комментарии/коммиты — английский.
- Next.js 16 непривычный — перед нетривиальными вещами читай `node_modules/next/dist/docs/` (см. `AGENTS.md`).

## 5. Запуск локально

```bash
npm install
cp .env.example .env   # заполнить значения
npm run dev            # http://localhost:3000
```
> На текущем прототипе БД ещё нет — он работает на in-memory данных. Подключение Postgres/Prisma — это Блок 1 плана.
